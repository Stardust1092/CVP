## 1. 架构概述

系统分为**浏览器端采集层**和**服务器端推理层**，通过一条持久 WebSocket 连接。

```
┌─────────────────────────────────────────────────────────┐
│  浏览器                                                  │
│                                                         │
│  getUserMedia ──► <video> ──► captureCanvas (15 fps)    │
│       │                             │                   │
│       │           JPEG 0.7 质量（WebSocket 二进制帧）    │
│       │                             │                   │
│  Web Speech ASR ──► JSON 文本 ──────┤                   │
│                                     │                   │
│  ◄── JSON 状态 ◄── WebSocket /ws ◄──┘                   │
│        │                                                │
│  Canvas 叠加层                                           │
│  （检测框 · 手部骨架 · 方向箭头）                         │
│                                                         │
│  Web Speech TTS ◄── 引导文本                             │
└──────────────────────────┬──────────────────────────────┘
                           │  ws://localhost:8000/ws
                           │  wss://<局域网IP>:8443/ws
┌──────────────────────────▼──────────────────────────────┐
│  FastAPI 服务器（app/main.py）                           │
│                                                         │
│  asyncio 事件循环                                        │
│    ws.receive() ──► 二进制 ──► ThreadPoolExecutor(1)     │
│                                   │                     │
│                           FrameProcessor                │
│                           ├─ ObjectDetector (YOLOv8n)   │
│                           ├─ HandTracker (MediaPipe)    │
│                           └─ GuidancePolicy             │
│                                   │                     │
│    ws.send_json() ◄───────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

**关键设计决策：**

| 决策 | 理由 |
|---|---|
| 浏览器拥有摄像头 | 无需服务器 USB/V4L 访问；支持移动端使用 |
| 二进制 WebSocket（而非 HTTP multipart） | 开销更低，无逐帧 HTTP 握手 |
| `ThreadPoolExecutor(max_workers=1)` | 序列化 YOLO + MediaPipe 调用，防止 GPU 显存竞争 |
| `asyncio.Lock` 背压机制 | 若前一帧仍在处理中则丢弃新帧，保持界面响应 |
| 归一化坐标（0–1） | 前端叠加层与分辨率无关，适配任意摄像头分辨率 |
| 自动生成自签名 TLS 证书 | 无需 CA 证书即可在局域网移动端启用 `getUserMedia` |

## 3. 数据流

### 后端逐帧处理流水线

```
JPEG 字节（WebSocket 二进制）
        │
        ▼
cv2.imdecode()  →  BGR ndarray（H×W×3）
        │
        ├──► ObjectDetector.detect(frame, target_class)
        │         YOLOv8n 推理（GPU）
        │         按目标 COCO 类别过滤
        │         返回置信度最高的检测框 (x1,y1,x2,y2) 或 None
        │
        ├──► HandTracker.track(frame)
        │         BGR → RGB 转换
        │         MediaPipe GestureRecognizer.recognize_for_video()
        │         返回 list[NormalizedLandmark]（21 个关键点）或 None
        │         副作用：存储手势标签（Open_Palm / Closed_Fist …）
        │
        └──► GuidancePolicy.update(hand_center, obj_bbox, hand_open, shape, target)
                  按优先级排序的状态转换（见第 7 节）
                  冷却时间检查  →  输出中文文本或 None
                  返回 (zh_text, cv_text)

状态字典（归一化坐标）  →  WebSocket JSON  →  浏览器
```

### 前端采集与发送

```
setInterval(_captureAndSend, 1000/15)   ← 15 fps 定时器
        │
        ▼
drawImage(videoEl)  →  _captureCanvas
        │
        ├── facingMode === 'user'（前置摄像头）:
        │       ctx.scale(-1, 1)  →  水平翻转
        │       （服务器收到翻转帧 → 左右引导与实物一致）
        │
        └── facingMode === 'environment'（后置摄像头）:
                直接绘制
        │
toBlob('image/jpeg', 0.7)  →  arrayBuffer  →  ws.send(buf)
```

> **镜像不变量：** `<video>` 元素在前置摄像头时通过 CSS（`scaleX(-1)`）镜像显示。Canvas 叠加层**不**做 CSS 镜像；服务器返回的坐标（基于翻转帧计算）可直接映射到镜像后的视频显示上。

## 4. 模块说明

### `app/config.py`

所有常量的唯一来源，无业务逻辑。

```python
YOLO_MODEL        = "yolov8n.pt"
DETECTION_CONF    = 0.40          # YOLO 置信度阈值
DETECTION_IOU     = 0.45          # NMS IoU 阈值
MAX_HANDS         = 1
HAND_DETECT_CONF  = 0.65
HAND_TRACK_CONF   = 0.50
XY_ALIGN_THRESHOLD   = 0.12      # 归一化帧比例
CLOSE_DISTANCE_RATIO = 0.65      # 相对于目标最长边
GUIDANCE_COOLDOWN_SEC = 2.5
ZH_TO_COCO: dict[str, str]       # 中文 → COCO-80 类名
GUIDANCE_TEXTS: dict[str, str]   # key → 中文 TTS 语句
CV_TEXTS: dict[str, str]         # key → 英文叠加文字
```

---

### `app/detector.py` — `ObjectDetector`

| 方法 | 签名 | 说明 |
|---|---|---|
| `detect` | `(frame, target_class?) → list[dict]` | 运行 YOLOv8n；按类别过滤；按置信度降序排列 |
| `draw_detections` | `(frame, detections, target_class?) → frame` | 在帧上绘制检测框（WebSocket 模式下不使用） |

检测结果字典格式：
```python
{ "class": "bottle", "conf": 0.87, "bbox": (x1, y1, x2, y2) }
```

---

### `app/hand_tracker.py` — `HandTracker`

使用 `GestureRecognizer`（MediaPipe Tasks）的 `VIDEO` 模式。通过 `max(perf_counter_ms, last_ts + 1)` 强制时间戳严格递增。

| 方法 | 返回值 | 说明 |
|---|---|---|
| `track(frame)` | `list[NormalizedLandmark] \| None` | 对单帧 BGR 图像推理；更新 `_last_gesture` |
| `get_palm_center(lm, shape)` | `(x, y)` 像素 | 腕关节 + 4 个掌指关节的均值 |
| `is_hand_open(lm)` | `bool` | 优先使用手势标签；回退到指尖展开度启发式判断 |
| `get_gesture_label()` | `str` | 最近检测到的手势名称 |

张开手势：`Open_Palm`、`Victory`、`ILoveYou`、`Pointing_Up`
握拳手势：`Closed_Fist`、`Thumb_Down`、`Thumb_Up`

---

### `app/guidance.py` — `GuidancePolicy`

有状态类。整个服务器生命周期内共享同一实例（跨 WebSocket 连接）。

| 方法 | 说明 |
|---|---|
| `update(hand_center, obj_bbox, hand_open, frame_shape, target_class)` | 计算下一条指令；返回 `(zh_text, cv_text)` 或 `(None, None)` |
| `get_direction_vector(hand_center, obj_bbox, frame_shape)` | 手部到目标的单位方向向量；已对齐时（模 < 0.02）返回 `None` |
| `reset()` | 目标变更时调用 |

**`_emit(key)`** — 遵循 `GUIDANCE_COOLDOWN_SEC`：相同指令仅在冷却结束后重复；不同指令立即触发。

---

### `app/camera_processor.py` — `FrameProcessor`

应用启动时创建的单例。通过 `threading.Lock` 保护目标状态，线程安全。

| 方法 | 线程安全 | 说明 |
|---|---|---|
| `set_target(text)` | ✓ | 中文→COCO 映射；重置引导状态 |
| `clear_target()` | ✓ | 重置状态 |
| `process_frame(jpeg_bytes)` | Executor 线程 | 完整处理流水线；返回状态字典 |
| `pop_guidance()` | ✓ | 出队一条引导文本（或 `None`） |
| `get_state()` | ✓ | 最近一帧处理结果的快照 |

状态字典键：`hand_detected`、`target_detected`、`hand_open`、`guidance_state`、`last_guidance`、`target_class`、`target_display`、`target_bbox_norm`、`hand_center_norm`、`hand_landmarks_norm`、`direction_norm`

---

### `app/main.py` — FastAPI 应用

**WebSocket 生命周期：**

```python
await ws.accept()
loop = asyncio.get_running_loop()
proc_lock = asyncio.Lock()

while True:
    raw = await asyncio.wait_for(ws.receive(), timeout=1.0)

    if raw["type"] == "websocket.disconnect":
        break                         # 正常退出

    if raw.get("bytes"):
        if proc_lock.locked():
            continue                  # 背压：丢弃帧
        async with proc_lock:
            state = await loop.run_in_executor(_exec, processor.process_frame, raw["bytes"])
        ...

except (WebSocketDisconnect, RuntimeError):
    pass
```

**为何用 `asyncio.get_running_loop()` 而非 `get_event_loop()`：** `get_event_loop()` 在 Python 3.12+ 中已废弃，在某些 executor 上下文中可能抛出异常。

## 5. WebSocket 协议

**端点：** `ws://host:8000/ws`（HTTP）· `wss://host:8443/ws`（HTTPS）

### 客户端 → 服务器

| 消息类型 | 格式 | 说明 |
|---|---|---|
| 视频帧 | `bytes`（JPEG） | 浏览器摄像头的原始 JPEG 帧 |
| 设置目标 | `text`（JSON） | `{"type": "set_target", "target": "瓶子"}` |
| 清除目标 | `text`（JSON） | `{"type": "clear_target"}` |

### 服务器 → 客户端

**`guidance`** — 有新指令时发送：
```json
{
  "type": "guidance",
  "text": "向右移动"
}
```

**`state`** — 每帧处理后发送：
```json
{
  "type": "state",
  "data": {
    "hand_detected": true,
    "target_detected": true,
    "hand_open": false,
    "guidance_state": "aligning",
    "last_guidance": "向右移动",
    "target_class": "bottle",
    "target_display": "瓶子",
    "target_bbox_norm": [0.31, 0.18, 0.62, 0.74],
    "hand_center_norm": [0.50, 0.50],
    "hand_landmarks_norm": [[0.48, 0.52], "...共 21 个"],
    "direction_norm": [0.71, 0.00]
  }
}
```

> 所有空间坐标均归一化到 `[0.0, 1.0]`，相对于帧的宽高。
> `direction_norm` 是从手部中心指向目标中心的单位方向向量 `[dx, dy]`；已对齐时（模 < 0.02）为 `null`。

**`target_confirmed`**：
```json
{"type": "target_confirmed", "display": "瓶子", "coco_class": "bottle"}
```

**`target_cleared`**：
```json
{"type": "target_cleared"}
```

## 7. 引导状态机

### 状态说明

| 状态 | 说明 |
|---|---|
| `idle` | 未设置目标 |
| `no_target` | 已设置目标但帧中未检测到 |
| `no_hand` | 已检测到目标，但帧中无手部 |
| `aligning` | 已检测到手部；引导左/右/上/下 |
| `approaching` | XY 方向已对齐；引导手部前进（深度方向） |
| `grasping` | 手部已足够靠近；指示张开/握拳 |
| `success` | 手部在目标处握拳——任务完成 |

### 转换逻辑（按优先级排序）

```
target_class 为 None
  └──► IDLE

obj_bbox 为 None
  └──► NO_TARGET  →  播报"请寻找目标"

hand_center 为 None
  └──► NO_HAND    →  播报"请伸出手"

|Δx_norm| > XY_ALIGN_THRESHOLD (0.12)
  └──► ALIGNING   →  播报"向右移动"或"向左移动"

|Δy_norm| > XY_ALIGN_THRESHOLD (0.12)
  └──► ALIGNING   →  播报"向下移动"或"向上移动"

dist_px ≥ obj_size × CLOSE_DISTANCE_RATIO (0.65)
  └──► APPROACHING →  播报"向前靠近"

hand_open == True
  └──► GRASPING   →  播报"握住目标"

hand_open == False（首次）
  └──► SUCCESS    →  播报"抓取成功"
```

其中：
- `Δx_norm = (obj_cx - hand_cx) / frame_W`
- `dist_px = 手部中心与目标中心的欧氏距离`
- `obj_size = max(obj_w, obj_h)`

## 8. 配置参考

所有配置均在 `app/config.py` 中，无环境变量或外部配置文件。

### 目标检测

| 参数 | 默认值 | 范围 | 说明 |
|---|---|---|---|
| `YOLO_MODEL` | `"yolov8n.pt"` | 任意 YOLO 权重 | `n`=最快，`s/m/l/x`=更精准 |
| `DETECTION_CONF` | `0.40` | 0.1–0.9 | 越低召回率越高，噪声越多 |
| `DETECTION_IOU` | `0.45` | 0.1–0.9 | NMS IoU 阈值 |

### 手部追踪

| 参数 | 默认值 | 说明 |
|---|---|---|
| `MAX_HANDS` | `1` | 仅在有多手引导逻辑时增大 |
| `HAND_DETECT_CONF` | `0.65` | 检测置信度（越高假阳性越少） |
| `HAND_TRACK_CONF` | `0.50` | 每帧追踪置信度 |

### 引导参数

| 参数 | 默认值 | 调优指南 |
|---|---|---|
| `XY_ALIGN_THRESHOLD` | `0.12` | 增大 → 手部偏离更多才触发引导。640px 时：0.12 → ±77px |
| `CLOSE_DISTANCE_RATIO` | `0.65` | 减小 → 抓取阶段更早触发（手离目标更远时） |
| `GUIDANCE_COOLDOWN_SEC` | `2.5` | 减小 → TTS 播报更频繁；增大 → 更安静但反馈更慢 |

### 添加物体映射

在 `ZH_TO_COCO` 中添加条目以支持新中文名称：
```python
ZH_TO_COCO: dict[str, str] = {
    "水壶": "bottle",
    "保温杯": "cup",
    # COCO-80 类名参考：https://github.com/ultralytics/ultralytics/blob/main/ultralytics/cfg/datasets/coco.yaml
}
```

## 9. 前端内部实现

### 摄像头与采集（`app.js`）

```
startCamera()
  ├── navigator.mediaDevices.getUserMedia({ video: {facingMode} })
  ├── videoEl.srcObject = stream
  ├── _applyMirror(facingMode === 'user')   ← 仅对 video 做 CSS scaleX(-1)
  ├── _unlockTTS()                          ← iOS/Android 音频解锁（需在用户手势内调用）
  └── startFrameCapture()                   ← setInterval @ 15 fps

_captureAndSend()
  ├── drawImage(videoEl)  [原始帧，忽略 CSS 变换]
  ├── 若为前置摄像头: ctx.scale(-1,1) 翻转  ← 服务器接收方向正确的帧
  └── toBlob(jpeg, 0.7) → ws.send(buf)
```

### Canvas 叠加层

`drawOverlay(state)` 在 `#overlay-canvas`（叠加于 `<video>` 上）绘制：
- **检测框** — 绿色矩形 + 角标 + 中心点 + 标签
- **手部骨架** — 21 个关键点 + HAND_CONNECTIONS 连线（绿色=张开，蓝色=握拳）
- **手部中心** — 掌心处圆圈
- **方向箭头** — 从手部中心指向目标中心的线段 + 箭头

> Canvas **不做** CSS 变换。服务器坐标（基于翻转帧）可直接对应到 CSS 镜像后的视频显示上。若对 Canvas 也做 CSS 镜像，所有叠加层会被二次翻转。

### TTS（Web Speech API）

```
speak(text)
  ├── 去重：若与最后一条入队内容相同则跳过
  ├── speechSynthesis.cancel()   ← 立即中止过期语音
  └── drainTTS()
        └── SpeechSynthesisUtterance(text)
              lang='zh-CN', rate=1.1
              优先选中文语音 → 回退到浏览器默认
              speechSynthesis.speak(utt)

_unlockTTS()    ← 在 startCamera()（用户手势上下文）中调用一次
  └── 播放静音语音（volume=0）以解锁移动端音频
```

**iOS/Android 说明：** `speechSynthesis.speak()` 在用户手势内首次调用前会被阻止。`_unlockTTS()` 在点击启动摄像头按钮时触发静音语音，以满足此要求。

### ASR（Web Speech API）

```
toggleASR()
  └── SpeechRecognition({ lang: 'zh-CN', interimResults: false })
        onresult → setTarget(transcript)   ← 发送 WS set_target 消息
```

## 10. 启动服务器

### `run.py` 命令行

```
python run.py [--https]
```

| 参数 | 默认 | 说明 |
|---|---|---|
| `--https` | 关闭 | 额外在 8443 端口启动 HTTPS 服务器（移动端摄像头必需） |

### 双服务器模式（`--https`）

```python
# HTTP 在 8000 端口（后台线程，独立事件循环）
threading.Thread(target=_run_server, kwargs={host, port=8000}).start()

# HTTPS 在 8443 端口（主线程，阻塞）
_run_server(host, port=8443, ssl_certfile, ssl_keyfile)
```

两个服务器共享同一 `app.main` 模块导入 → 共用同一 `FrameProcessor` 单例。

### TLS 证书

若 `cert.pem` / `key.pem` 不存在，通过 `cryptography` 库自动生成：
- RSA 2048，有效期 365 天
- SAN：`localhost`、`127.0.0.1`、检测到的局域网 IP

**移动端首次访问证书绕过：**
- **Android Chrome**：点击「高级」→「继续访问（不安全）」
- **iOS Safari**：点击「显示详细信息」→「访问该网站」→ 刷新页面

## 11. 性能与调优

### 基准（RTX 5070 Laptop，640×480 @ 15 fps）

| 阶段 | 典型延迟 |
|---|---|
| JPEG 解码 | ~1 ms |
| YOLOv8n（GPU） | ~8–12 ms |
| MediaPipe 手势识别（CPU XNNPACK） | ~15–25 ms |
| 引导计算 + 序列化 | < 1 ms |
| **端到端服务器** | ~25–40 ms |

### 低端硬件降低延迟

1. 降低采集帧率：修改 `app.js` 中 `const CAPTURE_FPS = 10`
2. 降低分辨率：在 `getUserMedia` 约束中设置 `width: { ideal: 320 }, height: { ideal: 240 }`
3. 切换更轻量模型：`YOLO_MODEL = "yolov8n.pt"`（已是最轻量版本）
4. 降低 JPEG 质量：`toBlob('image/jpeg', 0.5)` — 减少带宽，略微降低精度

### 背压机制

若新帧到达时 `proc_lock` 仍被持有，该帧会被直接丢弃（不入队）。这可防止内存无限增长，并确保界面始终反映最新摄像头状态。

## 12. 常见问题排查

### WebSocket 显示"连接中"（重连循环）

**现象：** 状态徽标始终不变绿；识别功能无效。
**原因：** WebSocket 处理器因 `RuntimeError: Cannot call "receive" once a disconnect message has been received` 崩溃。
**修复：** 已在 `main.py` 中修复——确认存在 `raw.get("type") == "websocket.disconnect"` 检查，且 `except (WebSocketDisconnect, RuntimeError)` 能捕获两者。

### 移动端摄像头无法启动

**原因：** `getUserMedia` 在非 localhost 的 HTTP 来源下被浏览器阻止。
**修复：** 使用 `python run.py --https` 启动，并通过 `https://<局域网IP>:8443` 访问。

### 移动端没有 TTS 语音

**原因：** iOS/Android 在用户手势调用前会阻止 `speechSynthesis.speak()`。
**修复：** `_unlockTTS()` 已在 `startCamera()`（按钮点击上下文）中调用。确保其在第一条引导消息到来之前已触发。

### 手部未被检测到

- 确保光线充足（MediaPipe 在极暗环境下效果差）
- 将手保持在画面中央 80% 区域内
- 若 `HAND_DETECT_CONF` 过高，可降低至 `0.50`

### MediaPipe 时间戳错误

**现象：** `Timestamp must be monotonically increasing`。
**修复：** 已在 `hand_tracker.py` 中修复——`ts_ms = max(perf_counter_ms - t0, last_ts + 1)`。

### Windows 终端中文乱码

```bash
set PYTHONIOENCODING=utf-8   # 在运行 python run.py 前执行
```

### 模型下载失败

手动将模型放入 `models/` 目录：
- `gesture_recognizer.task` — [从 Google 下载](https://storage.googleapis.com/mediapipe-models/gesture_recognizer/gesture_recognizer/float16/1/gesture_recognizer.task)
- `yolov8n.pt` — [从 Ultralytics 下载](https://github.com/ultralytics/assets/releases)
